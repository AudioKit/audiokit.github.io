Object Tree

Hierarchically, the structure of an AudioKit program usually looks like this:

  • Manager - which manages the sound engine and the creation of the...
  • Orchestra - which is a collection of...
  • Instruments - which has a series of interconnected...
  • Operations - each of which is controlled by their own...
  • Parameters - which can be audio streams, control signals, or constants and may be defined to be...
  • Properties - of either the instrument or the notes the instrument produces.

Instruments can also create notes which are controllable objects in their own right, following this flow:

  • Instruments can play instances of...
  • Phrases - which are collections of ...
  • Notes - each of which can have their own independent...
  • Properties - which define duration and local variations to the instrument's operations.

Installing Xcode Documentation

Code completion, pop-ups, and sidebar information are automatically created in Xcode, but if you would like Apple-style API reference documentation, download the docset:

Un-tar it and place it in the Shared Documentation folder:

{% highlight bash %} tar -xf ~/Downloads/io.audiokit.AudioKit.docset.tar.gz -C ~/Library/Developer/Shared/Documentation/DocSets/ {% endhighlight %}

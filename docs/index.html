--- layout: docs title: AudioKit Documentation header: Documentation nav_title: Docs permalink: /docs/ --- Hierarchically, the structure of an Audio Kit program usually looks like this:
  • Manager - which manages the sound engine and the creation of the...
    • Orchestra - which is a collection of...
      • Instruments - which has a descriptive set of...
        • Properties - which define the characteristics of a sound utilizing a plethora of available...
        • Operations - each of which is controlled by their own...
          • Parameters - which can be audio streams, control signals, or constants
Instruments can also create notes which are controllable objects in their own right, following this flow:
  • Instruments can spawn instances of...
    • Notes - each of which can have their own independent...
      • Properties - which define duration and local variations to the instrument's operations.

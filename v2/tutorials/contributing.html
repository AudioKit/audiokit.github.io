We’re really excited that you want to contribute to AudioKit! To help you get started, we’ve made this guide for contributing.

Fork AudioKit

First, you're going to want to fork a copy of the AudioKit repository to your own GitHub account. To do this, click on the “Fork” button in the top right corner of the repository page.

Next, you need to clone your forked repository to your computer. You can do this either though the terminal, or through a Git front-end, such as SourceTree. The screenshots below are from SourceTree.

Create a Branch

To add specific features, make a branch off your repository to test your ideas. Make sure you name your branch something descriptive so you can keep track of your work; this is especially important if you are working with multiple branches at once.

Run Tests

After you’ve finished working on your branch, run the test script to make sure your changes haven't broken anything. These tests will be automatically run on Travis when you make a pull request anyway, but the process is slow, so you might as well check locally first. To run the tests, you'll first need to make sure you have 'xcpretty' installed. If you don't, run the script below to install it:

{% highlight bash %} sudo gem install xcpretty {% endhighlight %}

Then, change your directory to the top level AudioKit directory and run:

{% highlight bash %} > ./Tests/travis.sh {% endhighlight %}

This will run tests and build projects and if all goes well the last line should read: {% highlight bash %} AudioKit passed validation. {% endhighlight %}

Make a Pull Request

Once you have confirmed that the tests pass locally, commit and push your changes.

Next, make a pull request on the AudioKit page on GitHub. Click on the button that says "Compare & pull request"

Then, click "Create pull request"

Switch Back to Main Development Branch

When your pull request has been approved, you'll see a notification in GitHub. At that point, change back to your main branch.

Then, delete the branch you were working on your changes in.

Finally, pull your changes from the develop branch. Make sure that you pull from “upstream”.

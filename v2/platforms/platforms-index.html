AudioKit works on iOS mobile devices and OSX computers. It is written in Objective-C but you can easily use it from a Swift-based app or application as well.

Audio Engine

AudioKit relies on a world-class core audio engine called Csound. We chose Csound for a variety of reasons which we outline below.

PropertyDescription
High Performance Core code is written in the C language and optimized for efficiency and speed.
Huge Library of Opcodes Csound has a library of opcodes that covers practically every facet of audio synthesis, effects, and analysis.
Open-Source Csound is completely open-sourced and always will be.
Cross-Platform Runs on Mac, Linux, and Windows as well as on iOS and Android devices.
Continuously Developed Csound is highly evolved, thanks to an excellent core team of developers dedicated to improving and modernizing the code.

What AudioKit provides on top of all this is a simple, human-readable interface to Csound's functionality within the familiar programming environment of Xcode and developing in Objective-C or Swift with integrated documentation and code-completion. You can easily tie your app's logic to sound or music parameters for instance. Additionally, you can use all of the modern features of these languages to govern audio events and sequences.

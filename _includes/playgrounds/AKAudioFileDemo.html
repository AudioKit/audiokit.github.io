AKAudioFile Demo

AKAudioFile inherits from AVAudioFile so you can use it just like any AVAudioFile :

{% highlight ruby %}import XCPlayground import AudioKit // Let's create an AKaudioFile : let ak = try? AKAudioFile(readFileName: "click.wav", baseDir: .Resources) // converted in an AVAudioFile let av = ak! as AVAudioFile //converted back into an AKAudioFile let ak2 = try? AKAudioFile(readAVAudioFile: av) {% endhighlight %}

The baseDirectory parameter if an enum value from AKAudioFile.BaseDirectory :

{% highlight ruby %}let documentsDir = AKAudioFile.BaseDirectory.Documents let resourcesDir = AKAudioFile.BaseDirectory.Resources let tempDir = AKAudioFile.BaseDirectory.Temp // So to load an AKAudiofile from this playground Resources folder : let drumloop = try? AKAudioFile(readFileName: "drumloop.wav", baseDir: .Resources) {% endhighlight %}

You can load a file from a sub directory like this:

{% highlight ruby %}let fmpia = try? AKAudioFile(readFileName: "Sounds/fmpia1.wav", baseDir: .Resources) {% endhighlight %}

As AKAudioFile is an optional, it will be set to nil if a problem occurs. Notice that an error message is printed in the debug area, and an error is thrown...

{% highlight ruby %}do { let nonExistentFile = try AKAudioFile(readFileName: "nonExistent.wav", baseDir: .Resources) } catch let error as NSError { print ("There's an error: \(error)") } {% endhighlight %}

So it's a good idea to check that the AKAudioFile is valid before using it. let's display some info about drumloop :

{% highlight ruby %}if drumloop != nil{ print("drumloop!.sampleRate: \(drumloop!.sampleRate)") print("drumloop!.duration: \(drumloop!.duration)") // and so on... } {% endhighlight %}

AKAudioFile can easily be trimmed and exported. First, you must set a callback function that will be triggered upon export has been completed.

{% highlight ruby %}func myExportCallBack(){ print ("myExportCallBack has been triggered. It means that export ended") if myExport!!.succeeded { print ("Export succeeded") // we get the resulting AKAudioFile let exportedfile = myExport?!.exportedAudioFile! // If it is valid, we can play it : if exportedfile != nil { print (exportedfile?.fileNamePlusExtension) let player = try? AKAudioPlayer(file: exportedfile!) AudioKit.output = player AudioKit.start() player!.play() } } else { print ("Export failed") } } {% endhighlight %}

Then, we can extract from 1 to 2 seconds of drumloop, as an mp4 file that will be written in documents directory. If the destination file exists, it will be overwritten.

{% highlight ruby %}let myExport = try? drumloop?.export("exported", ext: .m4a, baseDir: .Documents, callBack: myExportCallBack, inTime: 1, outTime: 2) {% endhighlight %}

AKAudioFile for writing / recording

AKAudioFile is handy to create file for recording or writing to. The simplest way to create such a file is like this:

{% highlight ruby %}let myWorkingFile = try? AKAudioFile() let mySecondWorkingFile = try? AKAudioFile() {% endhighlight %}

If you set no parameter, an AKAudioFile is created in temp directory, set to match AudioKit AKSettings (a stereo empty 32 bits float wav file at 44.1 kHz, with a unique name identifier :

{% highlight ruby %}if myWorkingFile != nil && mySecondWorkingFile != nil { let myWorkingFileName1 = myWorkingFile!.fileNamePlusExtension let mySecondWorkingFileName = mySecondWorkingFile!.fileNamePlusExtension } {% endhighlight %}

But the benefits of using AKAudioFile instead of AVAudioFile, is that you can normalize, reverse them or extract samples as float arrays. You can even perform audio edits very easily. Have a look to AKAudioFile Part 2...

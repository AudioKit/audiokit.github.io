We Love AudioKit and You Might Too!

Make no mistake about it, we built AudioKit out of a deep love for sound and music, and huge amount of respect for people who make amazing audio apps. Our passion for creating the best possible environment for creating audio apps has led us to build a toolkit that excels on numerous fronts, each of which are discussed in more-depth from the links on the left. To boil it down to a few points, AudioKit was built to be:

  • Easy-to-learn and understand
  • The quickest way to develop a real-world audio application
  • Powerful enough to enable your most ambitious ideas
  • Evolving and improving over time

Those goals are what have driven us to create a new version of AudioKit, version 3, which is a complete rewrite of AudioKit. The remainder of this page is dedicated to discussing how the features have changed from AudioKit 2 to 3.

Features where AudioKit 3 Crushes AudioKit 2

FeatureV2.xV3 Beta
Live Coding with Playgrounds

35 Objective-C Playgrounds based off of KZPlayground are powerful but do require extending Xcode with dynamic code injection and broke every time you updated Xcode.

AudioKit 3.0 brings audio development and experimentation to Xcode's native Swift Playgrounds. AudioKit Playgrounds allow for rapid prototyping of an audio system, with instant feedback of how your code sounds. You no longer have to recompile your project after changing a single parameter to hear what the new value sounds like-just make your changes, and the Playground will already have updated itself. Once you've written and tested your audio system using Playgrounds, you can use that code in your main app.

There are over 100 playgrounds included with AudioKit! The playgrounds come in two flavors, iOS, and OSX, because the playgrounds for each OS have different capabilities. OS X Playgrounds have the ability to spawn windows with sliders for instance. On the other hand, many processing and generation nodes don't work in OS X, so there is a trade-off.
Installation

Fairly easy installation as a static or dynamic library.

Extremely easy installation as a framework. Basically just drag the framework into your project. CocoaPods and Carthage support is planned and may be in place by the time you're reading this.
Examples

Hello World and AudioKitDemo.

Hello World plus Analog Synth X, a virtual analog synthesizer that is so cool we submitted it to the AppStore (as a free app of course!). AudioKit Particles and a few MIDI Sequencing demos are included as well.
Audio Engine

Csound

CoreAudio, AVAudioEngine and Soundpipe/Sporth which includes code from Chuck, Csound, Faust, Guitarix, ToneStack. Basically, if is open-sourced we can include it. Email us!
Integration w/ Audio Tools

Not so much.

AudioKit 3 attempts to be a cooperative member of any audio ecosystem. AudioKit's nodes are just Audio Units themselves, so they can be dropped in to a signal chain as you wish. Any third party tool can be easily included in the signal chain as well.

Sequencing

Powerful but the timing was imprecise.

Sequencing is achieved through Apple's CoreAudio sequencers, so it is quite powerful. There are few sequencing playrounds and examples included to get you up to speed.

Features where AudioKit 3 has a slight edge over AudioKit 2

FeatureV2.xV3 Beta
Docs and Tutorials

Gorgeous docs generated by AppleDoc. Good tutorials.

Gorgeous docs generated by Jazzy. Playgrounds do a great job of replacing tutorials.
Graphing

Plots of audio at input or output.

Plots of audio at any point in the signal chain.
Language

Written in Objective-C, accessible from Swift apps.

Written in Swift with C/C++ underpinnings. The framewowrk probably works fine with Objective-C projects, but we're focussing on Swift from now on, because it rocks!
License

MIT License with an asterisk.

MIT License. Period.

Features where AudioKit 2 still wins (but we're working on them)

FeatureV2.xV3 Beta
Sensible Defaults and Presets

Yes.

Defaults, yes. Presets not so much. But they are not off the radar and they will be added.
Stability

This code has been stable for a while and is powering apps in the Apple Store.

Well, AudioKit 3 still in Beta and is a complete rewrite of AudioKit, so we can't say for sure, but it's been working great for us so far.
Templates

Xcode templates for common classes.

No templates yet. While we're able to make templates, they are not too high on our priority list because AudioKit 3 is so flexible, you're much more free to code in your own style.
Tests

Extremely well tested.

Test infrastructure has been started, but no tests have been written yet. This is still very important to us and it is a high priority. Travis CI is implemented to ensure the frameworks compile and the projects can be built.

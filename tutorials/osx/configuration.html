There is a certain amount of configuration that is required on OSX that is not required to build iOS apps. The first step is to make sure the application knows where to search for the CsoundLib.framework file included within AudioKit. This is accomplished in the Build Settins Tab. In the screenshot below we use the $(SRCROOT) variable to refer to the source code root and move up a few directories to where we have AudioKit installed. Your path will be different depending on where you've downloaded AudioKit to relative to your project.

Then on the Build Phases tab, click the "+" button on the upper left of the window

Choose the "New Copy Files Phase"

Click the "+" on the bottom left

Find the framework from the pull-down menu within Libraries/OSX/csound-OSX:

Change the Destination to "Frameworks" so that your Copy Files section now looks like this screenshot:

Next, add another build phase by clicking on the "+" in the upper left, but this time choose the "New Run Script Phase":

Change the Shell to /bin/bash and past in the code:

{% highlight objective-c %} EXECFILE=${BUILT_PRODUCTS_DIR}/${EXECUTABLE_PATH} LIBPATH=${BUILT_PRODUCTS_DIR}/${PRODUCT_NAME}".app/Contents/Frameworks/CsoundLib64.framework/Versions/6.0" NEWLIBPATH="@loader_path/../Frameworks/CsoundLib64.framework/Versions/6.0" # space separated list of libraries TARGETS="CsoundLib64" for TARGET in ${TARGETS} ; do LIBFILE=${LIBPATH}/${TARGET} TARGETID=`otool -DX ${LIBPATH}/$TARGET` NEWTARGETID=${NEWLIBPATH}/${TARGET} install_name_tool -id ${NEWTARGETID} ${LIBFILE} install_name_tool -change ${TARGETID} ${NEWTARGETID} ${EXECFILE} done {% endhighlight %}

Congratulations, your application should now be set up to include everything it needs to run AudioKit.

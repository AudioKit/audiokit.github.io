There is a certain amount of configuration that is required on OSX that is not required to build iOS apps. The first step is to make sure the application knows where to search for the CsoundLib.framework file included within AudioKit. This is accomplished in the Build Settings Tab. In the screenshot below we use the $(SRCROOT) variable to refer to the source code root and move up a few directories to where we have AudioKit installed. Your path will be different depending on where you've downloaded AudioKit to relative to your project.

Then on the Build Phases tab, click the "+" button on the upper left of the window

Choose the "New Copy Files Phase"

Click the "+" on the bottom left

Find the framework from the pull-down menu within Libraries/OSX/csound-OSX:

Change the Destination to "Frameworks" so that your Copy Files section now looks like this screenshot:

Next, add another build phase by clicking on the "+" in the upper left, but this time choose the "New Run Script Phase":

Change the Shell to /bin/bash and past in this shell script:

{% highlight bash %} EXECFILE=${BUILT_PRODUCTS_DIR}/${EXECUTABLE_PATH} LIBPATH=${BUILT_PRODUCTS_DIR}/${PRODUCT_NAME}".app/Contents/Frameworks/CsoundLib64.framework/Versions/6.0" NEWLIBPATH="@loader_path/../Frameworks/CsoundLib64.framework/Versions/6.0" # space separated list of libraries TARGETS="CsoundLib64" for TARGET in ${TARGETS} ; do LIBFILE=${LIBPATH}/${TARGET} TARGETID=`otool -DX ${LIBPATH}/$TARGET` NEWTARGETID=${NEWLIBPATH}/${TARGET} install_name_tool -id ${NEWTARGETID} ${LIBFILE} install_name_tool -change ${TARGETID} ${NEWTARGETID} ${EXECFILE} done {% endhighlight %}

So that your run script phase looks like:

Finally, you'll need to delete any of the library folders we're not using. Since this is a Mac project, delete the iOS folder. Your Libraries folder should now look like the one below:

Because this is an Objective-C project, you'll also need to delete the Swift folder. If you were making a Swift-based Mac project, you would need to keep that folder in place.

Congratulations, your application should now be set up to include everything it needs to run AudioKit. You should now be able to follow the AudioKit tutorials for either Objective-C or Swift.

Setting up an OSX application to use Audio Kit is not as easy as it is for iOS, but still not too hard. After you’ve created a new project in Xcode, navigate to your AudioKit repository. Then, drag the folder that says “AudioKit” into your project.

After you do this, you’ll see a screen that will ask you to choose some options for adding the files. If you're just experiementing with Audio Kit for now, you can leave "Copy items if needed" unchecked. If you're making an app that will be sold on the App Store, then you'll want to make sure "Copy items if needed" is checked.

Then, click Finish.

After you’ve finished, you’ll see a folder on the left side of Xcode with AudioKit.

The next thing we need to do is delete the libraries that we won't be using for this project. Open up the AudioKit folder, you'll see a folder that says "Libraries" as shown below.

Since we're making an OSX project that uses Objective-C, we will not need the Swift or iOS folders.
